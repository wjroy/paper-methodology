2. Methodology
We constructed a framework for estimating LIBs SOH using fine-tuned pre-trained TimeGPT, as shown in Fig. 1. The framework consists of a fine-tuning component, pre-trained TimeGPT, and an inference process. The battery data used for fine-tuning and the details of the fine-tuning process will be elaborated in Section 2.2.
2.1. Framework overview
As the first foundational model for time series data, TimeGPT is trained on the Transformer architecture. It has been trained on over a trillion data samples, with the training set encompassing time series data from numerous domains, including finance, healthcare, weather, the Internet of Things, energy, and network traffic. These time series datasets span different lengths, periods, and sampling frequencies. However, the specific training data and parameters have not been publicly disclosed, and currently, it can only be accessed via an API call.

2.2. Fine tuning
Fine-tuning serves as a bridge between pre-trained models and specific downstream tasks. By re-training the model on the dataset of the target task, fine-tuning specializes the model parameters. In this study, fine-tuning plays a crucial role, and the entire process is divided into four stages. Specifically, these include data preprocessing, constructing the fine-tuning dataset by concatenating the data, fine-tuning with the input dataset, and inference by determining the prediction step length based on the test set.
Stage 1 involves preprocessing the data collected from six types of cathode materials, comprising a total of 143 LIBs. The corresponding SOH is calculated using the Coulomb counting method and used as the target value. In Stage 2, the preprocessed battery data from Stage 1 is concatenated to form the fine-tuning dataset. In Stage 3, the fine-tuning dataset obtained in Stage 2 is input into TimeGPT for fine-tuning.
