2. Methodology

The main purpose of this research is to develop a two-stage LSSVM framework for accurately perceiving the ground settlement occurrence and then predicting the settlement evolution. It is known that the ground settlement occurs when the supporting soil loses its load-bearing capabilities, while it does not necessarily start at the beginning of deep excavation.

Therefore, we will firstly develop the data-driven classification of settlement occurrence using SLP-LSSVM and then realize the prediction of the settlement evolution by CSA-LSSVR. Fig. 1 shows the proposed theoretical framework containing two key stages. The detailed implementation process is introduced below.

2.1. SLP-LSSVM classification of settlement occurrence under class imbalance and uncertainty

The first stage aims to distinguish two classes named the settlement non-occurrence and settlement evolution through identifying the hyperplane (also known as the boundary) between the two classes. Due to the nature of settlement development in the real project, there is a significant imbalance between the available measurements corresponding to the two classes of "settlement non-occurrence" NI and "settlement evolution" NP, where NI is far less than NP i.e. NI << NP.

This kind of severe class imbalance makes the LSSVM model inapplicable for correctly recognizing the boundary between two classes. To tackle this issue, a statistical learning process (SLP) incorporating the copula theory and unfair sampling strategy is developed, whose detailed introduction is given as follows.

2.1.1. Problem formulation of data imbalance

To begin with, a feature space for settlement prediction is carefully defined. M features about the monitoring item in the deep excavation project are taken into account, and the value of each feature is recorded in the N continuous days as the whole process of the deep excavation project. That is to say, a feature space is formed by N time series items, which can be expressed by a matrix F in the size of N × M.

Note that for simplicity, the term 'items' instead of 'time series items' will be used in the following discussions. In the first NI days, no settlement occurs, which means that the settlement value keeps constant. This phase can be termed as "settlement non-occurrence". Correspondingly, a negative class label (Class indicator = -1) is assigned to the first NI items that are in the first rows NI of matrix F.

Afterwards, the settlement will occur from the (NI+1)th day and grow gradually in the next Np days. It enters a new phase called "settlement evolution". Correspondingly, a positive class label (Class indicator = 1) is given to the next NP items that are the (NI+1)th row to the last row (the Nth row) of matrix F. Naturally, NI and Np satisfy Eq. (1).

{ NI + Np = N
{ NP >> NI         (1)

where NI is the number of items belonging to the negative class, Np is the number of items belonging to the positive class, and N is the total number of items.

Obviously, there is an imbalance issue between the positive and the negative class in the practice of deep excavation, while such a severe skew will, unfortunately, lead to great challenges in imbalanced classification. To deal with it, data sampling needs to be carefully designed to balance the class distribution. The goal of data sampling is to generate samples for the feature space subject to the class imbalance problem and the potential uncertainties. These generated samples help to train a reliable classifier to find out a hyperplane between the class of "settlement non-occurrence" and "settlement evolution".

2.1.2. A statistical learning process (SLP) for data sampling

For the purpose of handling both data imbalance and uncertainty, a new sampling method called the statistical learning process (SLP) is mathematically designed relying on probabilistic models, which has been illustrated in Fig. 2. It can act as a notable breakthrough to perform a more reasonable data sampling procedure and better describe the probabilistic nature of features. It shows the significant merits of adapting to the complex construction environment and dealing with measurement noise.

From the problem formulation, there are totally N time series items during the whole settlement procedure, and each item is actually the output of the real M features at each time step. At time i, the output of the real M features (x1i, x2i, ..., xMi) can be considered as the mean values of the real values of the M features at that time.

As given in Eq. (2), it can be considered as the mean values of the real M features at that time (μx1i, μx2i, ..., μxMi) and the associated correlated noise at that time (γx1i, γx2i, ..., γxMi). In particular, the noises in the M features are generally correlated due to the influence from the common physical factors and the values measured from common or similar methods.

Correspondingly, the output (x1i, x2i, ..., xMi) can be expressed as the joint probability density function f_X1i,X2i,...,XMi, as provided in Eq. (3). It is the superposition (represented by the symbol "⊕") of the joint probabilistic model of the measured M features at that time (μx1i, μx2i, ..., μxMi), f_μX1i,μX2i,...,μXMi, and that of the associated interdependent noise (γx1i, γx2i, ..., γxMi), f_γX1i,γX2i,...,γXMi.

(x1i, x2i, ..., xMi) = (μx1i, μx2i, ..., μxMi) + (γx1i, γx2i, ..., γxMi)    (2)

f_X1i,X2i,...,XMi ← f_μX1i,μX2i,...,μXMi ⊕ f_γX1i,γX2i,...,γXMi    (3)

Further, we can combine the N items (x1i, x2i, ..., xMi) into one N × M matrix X to represent the N time series items as a whole. That is, the ith row of the matrix is the vector (x1i, x2i, ..., xMi). Similarly, we can also formulate the other two N × M matrices, i.e. the matrix of the measured features μX and the matrix of the associated errors γX. Following Eq. (4) the matrix X is actually the sum of those two matrices:

X = μX + γX    (4)

For an intuitive understanding, the designed SLP incorporating model identification and data calibration is visualized in Fig. 2. The SLP process can be consecutively carried out to build the joint probabilistic model for all the N items. That is to say, when the model is identified based on both the engineering practice and the analysis of measured features, the model will be further calibrated using the measured features and their correlated noises estimated by the knowledge of copula description.

Noticeably, the proposed SLP contains two important techniques, including a copula-based relationship description of feature space and the newly developed unfair sampling strategy, which are explained in detail below.

(1) Copula-based relationship description of the feature space

It is worth noting that the monitoring data presented in the feature space may not be necessarily equal to the real values due to the existence of some noises in the measurement process. It is believed that the noise associated with the measured features is normally dependent with other features sharing similar observation methods. The ignorance of such noises in the feature space is inadequate to perform reliable data sampling and further to describe the real features. Therefore, an essential task for guaranteeing sampling quality is to appropriately describe the unavoidable noise in features owning similar meanings.

Herein, the copula function, acting as a simple and useful quantitative tool, is flexibly deployed with less computational burden. It is outstanding to provide a multivariate functional form for the joint distribution of random variables from a small amount of data [35]. That is to say, a copula is responsible for modeling the dependence structure between targeted features from the pre-specified parametric marginal distributions of features.

The basic procedure in the copula-based relationship description of features is presented in Fig. 1. From a mathematical view, copula follows the Sklar's theorem [36]. An important thing to mention is that the time concept is added in copula in this study taking the time-variant properties of the features into account. That is, copula functions are defined for different time series items in the feature space instead of one copula function for all the items.

Let Fi be the joint cumulative distribution function (CDF) for the noise item (γx1i, γx2i, ..., γxMi) at time i (i = 1,2,...,N), as expressed in Eq. (5). Accordingly, in the bivariate case, the joint CDF under copula can be defined by Eq. (6) to tie the noises associated with features together. F_θt(x1, x2) provides a precise bivariate dependence profile between features x1 and x2 at time t.

Fi(γx1i, γx2i, ..., γxMi) = C(F1i(γx1), F2i(γx2), ..., FMi(γxMi))    (5)

F_θi(γx1i, γx2i) = C(F1i(γx1), F2i(γx2); θi)    (6)

where C is the copula function to build and simulate dependent structures in (γx1i, γx2i, ..., γxMi), Fji(γxji) is the marginal distribution of γxji (j = 1,2,...,M), and θi denotes a parameter vector of the copula at the time i.

The crucial aspect comes from the selection and calibration of the proper copula function to better fit the on-site monitoring data. Since Gaussian copula and t copula are the two popular types of copula functions relatively easy to implement, we present them here for illustration purposes. Both of them belong to the elliptical copulas, whose density contours are elliptical with constant eccentricity [37].

The Gaussian copula is the copula of the multivariate normal distribution, while t copula is the copula underlying the multivariate Student's t distribution. The bivariate copula functions of these two types at the time i can be generally written as Eqs. (7) and (8), respectively.

C_Ga_θi(u1, u2) = ΦR(φi^(-1)(γx1), φi^(-1)(γx2))    (7)

C_t_θi(u1, u2) = tν,Σ(tνi^(-1)(γx1), tνi^(-1)(γx2))    (8)

where ΦR is the bivariate normal distribution function with a linear correlation matrix R, φ^(-1) is the inverse of the univariate Gaussian distribution function, tν,Σ is the bivariate Student's t distribution with a correlation matrix Σ in ν degrees of freedom. The subscript i represents the distribution function corresponding to the time i (i = 1,2,...,N).

(2) Unfair sampling strategy for the feature space

Apart from the copula-based relation description of the feature space, another key prerequisite of sampling is to define the joint probabilistic models as presented by Eq. (3) for all the N items, which helps to capture the probabilistic nature of the features. Based upon the established probabilistic model, issues that arise for effectively oversampling the minority class are twofold, since the minority is also the class of interest, especially from the perspective of the detection of the trigger of settlement occurrence.

One is how to tackle the class imbalance by a stochastic process relying on numerical simulations; while the other is how to design rules for generating samples with high proportions to be close to the hyperplane. The solution of these two questions contributes to formulating a novel unfair sampling strategy by taking full advantage of the time-variant joint probabilistic information of the features and classification decision hyperplane in the feature space. It serves as a sound basis for reaching a perfect or almost perfect classification performance.

• Task 1: A robust mathematical technique named Monte Carlo simulation is strongly recommended to handle the measurement uncertainties, generally called the noises associated with the measurements, since it is good at holding the uncertain properties of the measured data. Its core idea is to randomly sample n sets of some item of the input features according to the joint probability density functions derived from the joint probabilistic model. As a result, samples can be produced to achieve the same size of two classes for balancing class distribution. In short, the hybrid of Monte Carlo and copula paves a realistic way for oversampling under sufficient consideration of uncertainty and dependency within the feature space.

• Task 2: An unfair sampling strategy is put forward here, acting as an improved variant of the naïve random sampling. In order to more easily take two classes apart, it is necessary to more reliably synthesize samples that are located near the decision hyperplane. As a feasible solution, the density corresponding to each item (also known as the weight of the item to be selected for sample generation) can be carefully defined as an inverse proportion to the Euclidean distance of this item to the hyperplane, aiming to focus more on samples near hyperplane.

Although the hyperplane is unknown at this stage, the NIth item in the negative class (corresponding to the NIth day of the deep excavation procedure) and the first item in the positive class (the (NI+1)th item in the feature space corresponding to the (NI+1)th day of the deep excavation procedure) are the two items closest to the hyperplane in their respective class.

Therefore, the sampling density corresponding to each item di (1 ≤ i ≤ N) is proportional to the reciprocal of the distance of this item to the NIth item in the negative class and the first item in the positive class respectively, as expressed in Eq. (9). This defined unfair sampling rule has been depicted in Fig. 1, which is good at strengthening the importance of samples near the hyperplane and filtering noisy samples. In the end, it helps to notice and draw more informative samples in a more efficient and precise manner, which is deemed more compatible with machine learning algorithms.

di =    {
        { 1/||X(i,:) - X(NI+1,:)|| / Σj=NI+2 to N (1/||X(j,:) - X(NI+1,:)||)  for NI+1 < i ≤ N (positive class)

where X(i, :) denotes the ith item in the feature space (the ith row of the feature matrix) (1 ≤ i ≤ N), and ||X(i, :) - X(j, :)|| represents the Euclidean distance between the two items. It is obvious from the definition of di (1 ≤ i ≤ N) that the sum of the density in both negative class and positive class is equal to one, as provided in Eq. (10). That is, there will no class imbalance in the test and training dataset.

2.1.3. LSSVM classification

The least-squares support vector machine (LSSVM) is employed in the classification task to precisely detect the beginning of settlement occurrence. LSSVM can be understood as a least-square version of the classical SVM, which can exhibit the advantages of high computational speed and good generalization.

Considering a training vector V with two classes given in Eq. (11), LSSVM attempts to build a classifier by Eq. (12) and make classification decisions according to Eq. (13). The final decision hyperplane is in the form of w^T φ(xi) + b = 0, which can easily distinguish the binary classes of "settlement non-occurrence" and "settlement evolution".

Noticeably, the main difference between SVM and LSSVM is that LSSVM based on a statistical learning theory utilizes a least-squares linear system as a loss function [38]. Thus, Eq. (12) needs to be converted into an optimization problem defined in Eq. (14) that results from the squared loss function. Accordingly, w and b can be obtained by minimizing the upper bound of generalization error.

V = {(x1, y1), ..., (xl, yl)}, (x ∈ R^n, y ∈ {-1, 1})    (11)

y(x) = sign[ΣNi w^T φ(xi) + b]    (12)

{ ΣNi w^T φ(xi) + b ≥ 0, then y(xi) = 1
{ ΣNi w^T φ(xi) + b < 0, then y(xi) = -1    (13)

where x ∈ R^n indicates x is a data vector in n dimension, and each data sample has one class label from y ∈ {-1, 1}. sign() is the signum function. w stands for the adjusted weight vector. φ(·) is the nonlinear feature map that can map the original feature space into a higher-dimension space, b is the biased term.

min 1/2 w^T w + γ/2 Σi=1 to l e_i²
s.t. yi[w^T φ(xi) + b] = 1 - ei    (14)

where γ represents the regularization parameter to avoid an overfitting problem and it should be pre-defined. ei means the error variables. Generally, Eq. (14) can be solved using Lagrange multipliers.

After training, an LSSVM model SI can be built based on the training dataset. The next necessary stage is to check the classification performance by feeding the test dataset into the trained model SI. As a quantitative assessment, the classification error rate in the test dataset can be calculated by Eq. (15). If εSI is not larger than the predefined allowable error rate εa (e.g. 0.05 in the investigations), the unfair sampling process can be finished. Otherwise, 2ne extra samples (ne for each class) will be continuously generated following the rule of unfair sampling. The data sampling procedure repeats until the equation of εSI ≤ εa is satisfied. Such a loop has been demonstrated on the left side of Fig. 1.

εSI = Σk=1 to 2nt (SI(k) - Y(k)) / 2nt    (15)

where SI(k) and Y(k) denotes the class indicator of the kth sample in the test dataset from the model SI and the existing knowledge during the sampling respectively, and nt is the size of the test set of each class.

2.2. Interval-based CSA-LSSVR prediction of settlement evolution under multi-source uncertainties

Once the ground settlement occurs, the settlement situation will evolve continuously during the deep excavation process. To capture the trend of settlement evolution, the second stage enters to make a precise prediction of ground settlement evolution.

2.2.1. CSA-LSSVR prediction in two interval forms

An LSSVR model under the CSA optimizer (CSA-LSSVR) is constructed to handle the defined regression problem about settlement evolution prediction. Afterward, since uncertainty is ubiquitous in engineering practice, two types of prediction intervals are carefully designed. The interval form is generated by the bootstrapping principle, which provides a desirable solution to manage multi-source uncertainty, including model uncertainty and data uncertainty. For an intuitive understanding, the implementation process of CSA-LSSVR prediction in the form of intervals is provided in Fig. 3.

(1) CSA optimized-LSSVR model

Herein, the LSSVR algorithm, as a regression version of the classical SVR, is carried out to predict the ground settlement caused by deep excavation. Ground settlement varies with time because there are time-variant influential features. An LSSVR-based meta-model is constructed to describe the direct link (non-linear relationship) between the influential features and the objective.

Considering a training set with data samples in the length of n, the input data and output data can be expressed as xi ∈ R^n and yi ∈ r, where R^n means the n-dimensional vector space and r indicates the one-dimensional vector space. Similar to LSSVM, the LSSVR model can be formulated as Eq. (16) with a kernel function to provide the prediction results of ground settlement evolution. Due to a set of linear equations, LSSVR is superior in reducing computation complexity.

y(x) = Σi=1 to N αi K(x, xi) + b    (16)

where K is the kernel function with different types satisfying the Mercer's condition, and b is the bias term.

For bringing the predictions closer to the observations, an optimization algorithm named coupled simulated annealing (CSA) is combined with the LSSVR model to automatically configure optimal parameters for the given LSSVR. CSA can be understood as a set of parallel SA processes coupled by acceptance probabilities. Its principle is to use the acceptance temperature to control the variance of the acceptance probabilities with a control scheme.

It can lead to improved optimization efficiency because it reduces the sensitivity of the algorithm to the initialization parameters while guiding the optimization process to quasi-optimal runs. Compared with classical optimization techniques (i.e., simulated annealing, genetic algorithms, particle swarm optimization), CSA shows advantages of global search, faster convergence, and higher robustness [39].

(2) Interval generation with a bootstrapping method

Another thing to be noted is that a mixture of uncertainty characteristics will unavoidably exist in the CSA-LSSVR model. In this regard, it is meaningful to take into account two significant sources of uncertainties, including model uncertainty and data uncertainty. Both of them are possible to exert unconsidered impacts on the cause of prediction deviations from the ground truth.

For modeling the uncertainty, a potential solution is to perform the bootstrapping method for sampling without any parametric assumption. As a result, several training sets in the same size can be generated repeatedly with replacement by setting the size of the training set and the number of iterations. Since the idea of bootstrapping method has served as an applicable and powerful statistical tool for quantifying the uncertainty associated with a statistical learning method [40], it is helpful to easily estimate the prediction errors and confidence intervals for the prediction results of ground settlement. The process for describing two sources of uncertainty in the interval form is presented below.

The type 1 prediction interval only considers the first source of uncertainty named model uncertainty. It stems from the training dataset ΔT that is randomly separated from the whole dataset. Since it is known that the prediction performance of the LSSVR model greatly relies on the training set, the fundamental concept for portraying such uncertainty is to independently produce a bunch of different training sets and then build their corresponding LSSVR models.

Such a data preparation process prior to fitting the model can be understood as the bootstrapping method for sampling, where the model is trained on the bootstrapped data and then tested on the out of bag (OOB) data. In short, the bootstrapping process is implemented as follows:
(1) Set the iteration number as Q;
(2) Random select the training data with a sample size n from a vast collection of monitoring data and repeat it Q times;
(3) Build Q LSSVR models to show diverse prediction performances on the OOB test data;
(4) generate a series of outputs {y1, ..., yQ} as a reflection of model stability.

As for the type 2 prediction interval, it is expected to consider both the model uncertainty and data uncertainty. In particular, data uncertainty is attributed to poor measurements and complex underground environments in practice, which can be regarded as noise. The key point is to mathematically portray such noise before the bootstrapping process, which can be normally described by a normal distribution associated with the original data.

After adding noise, the remaining steps are similar to the investigations with model uncertainty. It is carried out in the prepared dataset owning noise and duplicates Q times underlying Bootstrap, resulting in a wide-ranging set of prediction results about the ground settlement {y1, ..., yQ}. In nature, this iterative process paves a more comprehensive way in checking the robustness of the LSSVR model to noise.

In the end, the bootstrapping process can give back a series of prediction results in the length of Q, which commonly can construct a prediction distribution for further statistical inference. When the iteration times are large enough, the Central Limit Theorem indicates that the prediction distribution is prone to be closely approximated to a normal distribution.

After that, two quantile regressions, including the 0.05 quantile as the lower boundary and the 0.95 quantiles as the upper boundary, are deployed to generate a 0.9 prediction interval. That is to say, there is a 90% probability of the actual observations falling within the prediction interval.

2.2.2. Prediction performance evaluation

Apart from the model parameter, there are two other things to directly influence the LSSVR model prediction performance. On the one hand, influential features closely related to the settlement evolution need to be carefully determined. Herein, the number of features in the regression task of settlement prediction (M') is not necessarily equal to that in the classification task of the settlement occurrence (M), because some features might be irrelevant to settlement evolution.

On the other hand, for the purpose of prediction and assessment, the entire dataset needs to be randomly separated into the training set and the test set. As a result, the model training can produce various LSSVR-based meta models Sp. The meta-models offer a description of settlement evolution, which can be expressed as the function of the M' features (X'1, X'2, ..., X'M') in Eq. (17). 2.2.3. Global sensitivity analysis

To further enhance the understanding of the intelligent model, global sensitivity analysis (GSA) is performed to measure the influence of the features on the ground settlement prediction. The mathematical explanations from GSA assist in easily gaining access to the non-linear input-output relationships in CSA-LSSVR. Unlike the local sensitivity analysis, GSA can not only take into account the interactions among inputs with regard to the whole input feature space, but also provide a quantitative measure of the relative contributions of an individual feature to the overall output variance. Due to these superiorities of GSA, it has been more popular in geotechnical engineering [41,42].

Herein, three widely-used GSA methods are performed in the established LSSVR model, including the Elementary Effect Test (EET, also known as Morris) [43], the Sobol method [44], and Fourier Amplitude Sensitivity (FAST) [45]. All three methods focus on the variance of the model output to recognize the key features under the uncertain and complex underground, which are presented below. As expected, GSA can identify features with high sensitivity values as the most critical ones in controlling the ground settlement. It supports managers to formulate more credible decisions in keeping the stability of deep excavation.
